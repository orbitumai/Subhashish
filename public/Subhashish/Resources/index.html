Educational Guide — Claude.ai + Claude Code

Claude Commands & Token Optimization

Everything you need to use Claude efficiently — slash commands, model selection, prompt patterns, and resources — in one place.

What are slash commands? Type these directly in Claude Code or the Claude.ai interface. They give you precise control over sessions, context, and token usage without writing extra prompts.

Session Control

/compact
Compress conversation history
Summarizes older messages while keeping important context. Frees up context window space mid-session without losing your thread.
Use every 20–30 min in long sessions
/clear
Wipe context, start fresh
Clears the entire conversation history. Best used when switching to a completely new task — prevents context drift.
Most underused command by beginners
/cost
Show token usage
Displays input tokens, output tokens, and estimated cost for the current session. Helps you track where tokens are going.
API users: check /cost often. Subscribers: use claude.ai → Settings → Usage

Content & Review

/review
Targeted code review
Requests a structured review of recent changes. More reliable than asking "review my code" freeform — produces consistent, actionable output.
/init
Initialize project memory
Creates a CLAUDE.md file for your project. This file loads before every session — keep it lean (under 500 tokens) or it costs you every single turn.
Run this first in every new project
/copy
Copy assistant response
Copies the latest response to clipboard. Use /copy N to copy the Nth-latest assistant response.

Model & Configuration

/model
Switch models mid-session
Change the active Claude model without starting a new session. Drop to Haiku for simple tasks, escalate to Opus for complex reasoning.
Biggest token lever available
/effort
Set thinking depth
Controls extended thinking budget. Levels: low → medium → high → xhigh (Opus 4.7 only). Lower effort = fewer thinking tokens consumed.
/help
List all available commands
Shows every slash command available in your current session, including any custom commands you've created.

Custom Commands (Claude Code)

.claude/
Create your own slash commands
Save Markdown files in .claude/commands/ (project) or ~/.claude/commands/ (global). The filename becomes the command name. These are reusable prompt shortcuts — ideal for no-code builders.
Example: /orbit, /review-copy, /pitch-check

The golden rule: tokens burn in three places — what you send in, what Claude generates, and what's loaded from memory (CLAUDE.md, system prompts). Control all three.

Top Habits

↗
Plan in Chat, build once
Explore ideas in chat before creating artifacts (docs, spreadsheets, slides). Artifact generation costs significantly more tokens.
@
Use @file, not paste
Reference files with @filename instead of pasting their content. Claude tokenizes file mentions more efficiently than inline text dumps.
◻
One task per session
Start a fresh session for each new task. Mixing unrelated work in one session accumulates context that slows and costs more over time.
⬡
Compact before big tasks
Run /compact before starting a long or complex task in an existing session. Buys you more clean context window space.
⚡
Match model to difficulty
Use Haiku for drafts, rewrites, and simple questions. Save Opus for architecture, debugging, and multi-step planning. Most tasks don't need Opus.
◈
Keep CLAUDE.md lean
Your CLAUDE.md loads before every single session. A 5,000-token CLAUDE.md costs 5,000 tokens before you've typed a word. Trim it ruthlessly.

Token Cost by Action (Relative)

Action Token Cost Tip
Chat message (short) Very low Default for exploration
@file reference Low Always prefer over pasting
Code generation Medium Specify exactly what you need
Creating artifacts (docs/slides) Medium-high Plan in chat first
Extended thinking (high/xhigh) High Use /effort low for most tasks
Image generation High Only when you have a clear final vision

Model choice is the single biggest token lever. Haiku costs a fraction of Opus. For most no-code workflows, Sonnet 4.6 is the right default — only escalate to Opus for genuinely complex reasoning.

How to Switch

/model
Switch mid-session in Claude Code
Type /model and select from the picker. Applies immediately to the current session without losing history.
claude.ai
Model selector in chat UI
On claude.ai, use the model dropdown at the top of any conversation. Change per-conversation or set a default in Settings.

As a no-code builder, your prompt is your code. Precise prompts reduce back-and-forth, which means fewer total tokens spent reaching the same result.

Before and After Examples

Context — writing a landing page
Vague (costs more rounds)
Write a landing page for my product.
Precise (one-shot result)
Write a 3-section landing page for AILeadCalling, a voice AI tool for SMB outreach. Hero: hook + CTA. Section 2: 3 pain points with solutions. Section 3: social proof placeholder. Tone: confident, no hype. Output HTML.
Context — debugging an n8n workflow
Vague
My workflow isn't working, can you help?
Precise
My n8n HTTP Request node is returning a 401. The endpoint is POST https://api.example.com/leads. I'm passing Bearer token in the header. Here's the node config: [paste]. What's wrong?
Context — generating content
Vague
Write me a LinkedIn post about AI.
Precise
Write a LinkedIn post (150 words) from the POV of an AI founder who left Fortune 500 to build no-code AI tools. Hook: a counterintuitive insight about prompting. End with a question. Voice: direct, no buzzwords.

Prompt Structure That Saves Tokens

State the output format upfront
Tell Claude exactly what you want: "Output a JSON object", "Return only the HTML, no explanation". Prevents verbose preamble.
Set constraints explicitly
Word count, tone, structure, what to exclude. "No bullet points", "under 200 words", "skip the intro". Reduces iteration rounds.
Include a negative example
Tell Claude what NOT to do. "Don't use corporate jargon", "Don't repeat the brief back to me". Saves a correction round.
Ask for one thing at a time
Multi-part requests confuse priority. Break complex tasks into sequential prompts — each cheaper and more accurate than one giant request.
