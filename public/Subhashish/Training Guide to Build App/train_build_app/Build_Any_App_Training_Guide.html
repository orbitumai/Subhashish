💡 The Methodology

Understanding the why before the how. This methodology works for any app because it's based on principles, not just tools.

The Core Principle

✨ The Golden Rule

"Your job is to write prompts that make AI write correct code. Not to write the code yourself."

This is a fundamental mindset shift. You're not learning to code - you're learning to:

  • Architect - Design the structure before coding
  • Direct - Tell AI what to build with clear instructions
  • Validate - Test that the code does what you intended
  • Debug - Understand errors and fix them (with AI help)

Strategy First. Prompt Second. Results Always.

This is the MeetOrbi mantra, and it applies to any app:

1. Strategy First

Before writing any prompt:

  • What feature are you building?
  • What does success look like?
  • What are the constraints?
  • What files will be created/modified?

2. Prompt Second

Then write a structured prompt:

  • CONTEXT: What you just finished
  • TASK: The ONE thing to build
  • CONSTRAINTS: Rules and requirements
  • ACCEPTANCE: How to verify it works

3. Results Always

Validate before moving on:

  • Run the code
  • Test the feature
  • Fix any errors
  • Document what you built

The Prompt Template

Every Cursor prompt should follow this structure:

CONTEXT: [What you just finished - gives AI continuity]

TASK: [The ONE feature to build - keep focused]

CONSTRAINTS:
  - Tech stack: Next.js 14 + Supabase + TypeScript
  - Follow naming conventions from .cursorrules
  - Use Shadcn/ui components
  - Add proper error handling
  - Include TypeScript types

FILES TO CREATE/EDIT: [Exact file paths]

ACCEPTANCE:
  1. [First success criteria]
  2. [Second success criteria]
  3. [How to test it works]

The AI Development Loop

Step 1: Plan

Break feature into smallest possible unit (one prompt = one feature)

Step 2: Prompt

Write structured prompt with context, task, constraints

Step 3: Review

Read AI-generated code, understand what it does

Step 4: Test

Run the code, verify it works as expected

Step 5: Iterate

If broken, copy error → ask AI to fix → repeat until works

Step 6: Document

Update build log, note what works, move to next feature

Key Principles

Principle 1: One Prompt, One Feature ▼

Never try to build multiple features in one prompt. It confuses the AI and makes debugging impossible.

❌ Bad: "Build authentication, database schema, and homepage"

✅ Good: "Build Google OAuth authentication flow"

Principle 2: Test Before Moving On ▼

Never run Prompt 2 until Prompt 1's code works. Broken code compounds.

Validation checklist after every prompt:

  • Code runs without errors
  • Feature works as expected
  • No console errors in browser
  • Database updates correctly (if applicable)
Principle 3: Constraints Are Your Friend ▼

The more constraints you give AI, the better the output. Don't just say "build a button" - specify:

  • Which component library? (Shadcn/ui)
  • What colors? (from design system)
  • What happens on click? (specific function)
  • Where does it go? (exact file path)
Principle 4: Read Before You Run ▼

Always review AI-generated code before accepting it. Look for:

  • Does it match your constraints?
  • Are there obvious security issues? (hardcoded secrets, no auth)
  • Does it use the right libraries/imports?
  • Is error handling present?

You don't need to understand every line, but you should understand the structure.

When AI Gets It Wrong

AI will make mistakes. Here's how to handle it:

Scenario 1: Code Doesn't Run

Action:

  1. Copy the full error message
  2. Open Cursor Chat (Cmd+L)
  3. Paste: "This code failed with [error]. Fix it."
  4. Apply the fix
  5. Test again

Scenario 2: Code Runs But Wrong Behavior

Action:

  1. Describe expected vs actual: "Should do X but does Y"
  2. Ask Cursor: "Debug why this is happening"
  3. Review the fix, understand the mistake
  4. Update your prompt to prevent this in future

Scenario 3: AI Hallucinated APIs/Libraries

Action:

  1. Check package.json - does the library exist?
  2. Google the function name - does it exist in that library?
  3. Ask Cursor: "This function doesn't exist in [library]. Use the correct API."

Success Metrics

You're doing this right when:

  • 80%+ of generated code works first try
  • You can explain what each file does (even if not line-by-line)
  • Fixing errors takes minutes, not hours
  • You're building features in hours, not days
  • You can deploy to production with confidence

Tool #1: Cursor

What It Is

Visual Studio Code fork with built-in AI (Claude Opus/Sonnet) that writes code as you describe what you want.

Why We Use It

  • AI Composer (Cmd+I): Write multi-file features from one prompt
  • AI Chat (Cmd+L): Ask questions about your code
  • Inline AI (Cmd+K): Edit code with natural language
  • Codebase Context: AI understands your entire project

Cost

$20/month (Pro plan - required for best models)

Alternatives

GitHub Copilot ($10/mo), Codeium (free), VS Code + Continue extension (free) - but Cursor is the best for prompt-driven development

Tool #2: Claude Code

What It Is

Terminal-based AI agent that can execute entire coding tasks autonomously - runs commands, edits files, debugs errors.

Why We Use It

  • Autonomous Execution: "Set up authentication" → it does everything
  • Multi-Step Tasks: Handles complex workflows (install → configure → test)
  • Error Recovery: If something breaks, it debugs and fixes automatically
  • Best for: Setup tasks, migrations, repetitive changes

Cost

Pay-per-use (charges for Claude API tokens, ~$0.10-1.00 per task)

When to Use

Use Claude Code for: initial setup, database migrations, bulk refactoring. Use Cursor for: building features, UI work, iterative development.

Tool #3: Next.js

What It Is

React framework with built-in routing, server-side rendering, API routes, and optimizations.

Why We Use It

  • File-based routing: Create app/about/page.tsx → route exists at /about
  • Server Components: Fetch data on server (faster, more secure)
  • API Routes: Backend endpoints without separate server
  • Production-ready: Used by Netflix, Uber, TikTok
  • Vercel-optimized: Deploy in seconds

Cost

Free (open source)

Learning Curve

You don't need to know React - Cursor will write it for you. You just need to understand the folder structure.

Tool #4: Supabase

What It Is

Open-source Firebase alternative - gives you PostgreSQL database + authentication + file storage + real-time subscriptions.

Why We Use It

  • PostgreSQL: Industry-standard SQL database (not proprietary)
  • Row Level Security: Database-level permissions (ultra secure)
  • Built-in Auth: Email, OAuth, magic links out of box
  • Storage: Upload/serve files (avatars, documents)
  • Realtime: Live updates (chat, notifications)
  • Generous Free Tier: 500MB database, 1GB storage, 50MB file uploads

Cost

Free → $25/mo (Pro plan when you outgrow free tier)

Alternatives

Firebase (Google), PlanetScale (MySQL), Neon (Postgres) - but Supabase has best free tier + all features integrated

Tool #5: Vercel

What It Is

Cloud platform that deploys your Next.js app globally in seconds. Made by the creators of Next.js.

Why We Use It

  • Push to Deploy: Commit to GitHub → auto-deploys
  • Preview URLs: Every branch gets its own URL for testing
  • Global CDN: Fast everywhere in the world
  • Serverless Functions: API routes scale automatically
  • Zero Config: Detects Next.js, sets everything up

Cost

Free → $20/mo (Pro plan for custom domains + better limits)

Alternatives

Netlify, Railway, Render - but Vercel is best for Next.js (they invented it)

Tool #6: GitHub

What It Is

Version control platform - stores your code, tracks changes, enables collaboration.

Why We Use It

  • Version History: Undo any change, go back in time
  • Backup: Your code is safe in the cloud
  • Vercel Integration: Required for auto-deploy
  • Industry Standard: Every developer uses it

Cost

Free (unlimited private repos)

Git Basics You Need

git add .                  # Stage all changes
git commit -m "message"    # Save changes with description
git push                   # Upload to GitHub
git pull                   # Download latest changes

How They Work Together

Step 1: Development

Write prompt in Cursor → AI generates Next.js code → Test locally

Step 2: Database

App talks to Supabase (PostgreSQL) → Data stored, users authenticated

Step 3: Version Control

Commit code to GitHub → Changes tracked, backed up

Step 4: Deployment

Vercel detects GitHub push → Builds app → Deploys globally

Step 5: Production

Users access your app at yourapp.vercel.app or custom domain

Once set up, this entire pipeline runs automatically. Push code → Deployed in 30 seconds.
